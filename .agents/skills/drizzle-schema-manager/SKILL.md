# Drizzle Schema Manager - Instructions & Guidelines

Use this skill when you need to create, update, or verify the database schemas, run migrations, or write database queries using Drizzle ORM in the DevPlanr project.

## 1. Schema Definition Guidelines

Drizzle schemas are defined in [src/lib/server/db/schema.ts](file:///C:/Users/kueoi/Downloads/Rendi_/RENDI/Portofolio/ai_spec/DevPlanr-app-structure/src/lib/server/db/schema.ts) (or its splits) and MUST mirror [DATABASE_SCHEMA.sql](file:///C:/Users/kueoi/Downloads/Rendi_/RENDI/Portofolio/ai_spec/DATABASE_SCHEMA.sql) exactly.

### Table Definitions Mapping:
- **`users_profile`**: Holds user profiles linked to Supabase Auth `auth.users(id)`.
- **`projects`**: Holds user project details. The `status` field must be one of: `'draft'`, `'generating'`, `'ready'`, `'failed'`, `'archived'`.
- **`project_inputs`**: Holds input details (raw idea, target users, features, preferences, etc.) linked 1:1 with a project.
- **`generated_documents`**: Holds markdown/json documents generated by AI. Types: `'product_overview'`, `'prd'`, `'user_stories'`, `'database_schema'`, `'api_endpoints'`, `'task_breakdown'`, `'ai_agent_prompt'`, `'test_cases'`, `'sprint_estimation'`.
- **`generation_runs`**: Logs AI generation sessions, tokens count, models, and errors.
- **`project_templates`**: Pre-seeded project templates for users to select.

### Drizzle Model Example
```typescript
import { pgTable, uuid, text, timestamp, integer, pgEnum, jsonb, boolean, uniqueIndex } from 'drizzle-orm/pg-core';

// Define status enum or check constraints
export const projectStatusEnum = pgEnum('project_status', ['draft', 'generating', 'ready', 'failed', 'archived']);

export const projects = pgTable('projects', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').notNull(), // References auth.users
  name: text('name').notNull(),
  description: text('description'),
  appType: text('app_type'),
  platform: text('platform'),
  status: text('status').default('draft').notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow(),
});
```

## 2. Row Level Security & Query Enforcement

Supabase RLS is strictly active. All queries executing in routes/endpoints must filter results by the owner's `user_id`.

> [!IMPORTANT]
> Never query the database without asserting `user_id = auth.uid()` or the authenticated user's ID resolved from the request session.

### Query Pattern:
```typescript
import { db } from '$lib/server/db';
import { projects } from '$lib/server/db/schema';
import { eq, and } from 'drizzle-orm';

// Querying projects for authenticated user
const userProjects = await db
  .select()
  .from(projects)
  .where(
    and(
      eq(projects.userId, sessionUserId),
      eq(projects.status, 'ready')
    )
  );
```

## 3. Running Migrations

When schema changes are made:
1. Generate migration files using Drizzle Kit:
   ```bash
   npx drizzle-kit generate
   ```
2. Apply migrations locally or to the Supabase instance:
   ```bash
   npx drizzle-kit migrate
   ```
3. Verify that the new database tables, fields, constraints, and indexes correspond exactly to both the Drizzle schema and the local database state.
